What CSMs Need to Keep Updated

Most pillars are fully automated. CSMs only need to actively manage three things — plus one critical handoff step.

Customer Engagement
Risk Flags
CX Health Score Segment

MANUAL

MANUAL

MANUAL
• Update Meeting Cadence when it changes
• Create Risk Flag records when risk is identified
• Set once per account: SMB / MM / Enterprise / Alleva / Implementation / Billing CRM / iVerify / IQ Only
• Update Customer Sentiment after every key interaction
• Update status: Active → Recovering → Resolved as situation progresses
• Update if the customer's segment changes
• Log Events with Type = CX Economic Buyer Meeting for every EB interaction
• A score of 0.10 must always have an active escalation plan
• Blank defaults to MM (CX) weighting

AUTO
Support, Product Usage, Financial  —  All fully automated. No CSM action required for data entry. Low scores should trigger investigation and conversation. Escalate Financial score issues to Finance / Collections.

⚠  Critical — Implementation Handoff:
When an account completes implementation and is handed off to CSM, you must update CX Health Score Segment to the correct post-implementation value (SMB, MM, Enterprise, etc.) immediately. Leaving it as 'Implementation' will result in an incorrect health score.
